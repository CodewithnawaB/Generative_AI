🧠 Generative AI & Foundation Models

Page 1

1. What is Generative AI?

Definition

Generative AI is a type of Artificial Intelligence that can CREATE new content — text, images, audio, video, and code by learning patterns from massive datasets.

✅ Types of Gen AI

  • Text Generation (ChatGPT, Claude)
  • Image Generation (DALL-E, Midjourney)
  • Audio Generation (ElevenLabs)
  • Video Generation (Sora, Runway)
  • Code Generation (GitHub Copilot)
  • Multimodal (Gemini)

✅ Why Gen AI Exists

  • Save time on repetitive tasks
  • Create content automatically
  • Solve complex problems
  • Assist humans in every field
  • Automate business workflows
  • Enable new applications
How Gen AI Works: INPUT (Prompt) → AI MODEL → OUTPUT (Generated Content) "Write a story" → [Gen AI] → "Once upon a time..." Steps: Data → Training → Model → Prompt → Inference → Output → Feedback

2. What is a Foundation Model?

Definition

A Foundation Model is a very large AI model trained on massive data that serves as a BASE for many different tasks. Like the foundation of a building — you build many things on top of it.

ModelCompanyType
GPT-4OpenAIClosed
ClaudeAnthropicClosed
GeminiGoogleClosed
LLaMA 3MetaOpen Source
MistralMistral AIOpen Source
Foundation Model Pipeline: STEP 1: Pre-Training → Massive Dataset → Self-supervised Learning STEP 2: Architecture → Transformer → Attention → Billions of Parameters STEP 3: Fine-Tuning → Domain specific data → Expert model STEP 4: RLHF → Human feedback → Better & safer STEP 5: Deployment → API → Users access via apps

3. What is LLM?

LLM = Large Language Model — A very large AI model trained on massive text data to understand and generate human language.

LLM Internal Steps: STEP 1: Tokenization → "Hello World" → ["Hel","lo","Wor","ld"] STEP 2: Embedding → Each token → numbers [0.23, 0.87, 0.45...] STEP 3: Attention → Focus on important words in context STEP 4: Transformer → Many layers process meaning deeply STEP 5: Output → Predict next best token/word

🔗 LangChain & Prompts

Page 2

1. What is LangChain?

Definition

LangChain is a framework (toolkit) that helps developers build AI-powered applications using language models. It connects AI models with tools, databases, and logic.

Analogy: Foundation Model = Engine of a car 🚗 LangChain = Full car (steering, wheels, GPS, everything) LangChain Components: Models → Prompts → Chains → Memory → Agents → Tools → Vector Stores

2. What is a Prompt?

Definition

A Prompt is your INPUT or INSTRUCTION that you give to AI. It tells the AI what to do, how to respond, and what format to use.

TypeDescriptionUse When
Zero ShotNo examples — ask directlySimple questions
Few ShotGive examples first then askNeed specific format
Chain of ThoughtAsk AI to think step by stepMath / Logic problems
System PromptInstructions at start to control behaviorControl AI personality
Role PromptGive AI a role to playNeed expert knowledge
Template PromptReusable prompt with variablesRepeated tasks

⭐ Good Prompt Formula

ROLE + TASK + CONTEXT + FORMAT + EXAMPLE = Perfect Prompt

# Prompt Types Code Example from langchain_core.prompts import ChatPromptTemplate, FewShotPromptTemplate # Simple ChatPromptTemplate chat_prompt = ChatPromptTemplate.from_messages([ ("system", "You are expert {role}"), ("human", "Explain: {topic}") ]) # Few Shot Prompt examples = [ {"input": "happy", "output": "sad"}, {"input": "hot", "output": "cold"}, ]

3. Prompt Engineering Tips

❌ Bad Prompts

  • "Write something"
  • "Fix this"
  • "Make it good"
  • "Explain AI"

✅ Good Prompts

  • "Write 5 points about Python"
  • "Fix this Python error and explain why"
  • "Make it professional and formal"
  • "Explain AI for a beginner in 5 points"

⛓️ Chains & Memory

Page 3

1. What is a Chain?

Definition

A Chain connects multiple steps together in a sequence. Like an assembly line — each step passes its output to the next step.

Chain TypeHow it WorksBest For
Simple ChainPrompt → LLM → OutputBasic Q&A
Sequential ChainChain1 → Chain2 → Chain3Multi-step tasks
Router ChainDecides which chain to useDifferent topics
Transform ChainClean data then send to LLMData processing
RAG ChainSearch docs + LLM answerDocument Q&A
Conversation ChainChain with memoryChatbot
# Simple Chain using | pipe operator from langchain_ollama import ChatOllama from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser model = ChatOllama(model="llama3") prompt = ChatPromptTemplate.from_template("Explain {topic} simply") chain = prompt | model | StrOutputParser() result = chain.invoke({"topic": "AI"})

2. What is Memory?

Definition

Memory allows AI to remember previous conversations so it can give context-aware responses — like a human remembering a conversation.

Memory TypeWhat it StoresBest For
BufferMemoryALL messagesShort conversations
WindowMemoryLast N messages onlyLong conversations
SummaryMemorySummarizes old chatsVery long chats
TokenBufferMemoryBy token limitToken management
VectorStoreMemoryIn vector databaseSemantic search
# Memory in Chatbot from langchain.memory import ConversationBufferMemory from langchain_core.prompts import MessagesPlaceholder memory = ConversationBufferMemory(return_messages=True) memory.save_context( {"input": "My name is Ahmed"}, {"output": "Hello Ahmed!"} ) history = memory.load_memory_variables({})
Memory Flow: User: "My name is Ahmed" Bot: "Hello Ahmed!" User: "What is my name?" Bot: "Your name is Ahmed!" ← Memory works! ✅

🕵️ Agents & Tools

Page 4

1. What is an Agent?

Definition

An Agent is an AI that can make decisions, use tools, and take multiple steps to solve complex problems. Unlike a simple chain, an Agent THINKS about what to do next.

📚 RAG (Simple)

  • Searches documents only
  • One path always
  • No decision making
  • Cannot use tools
  • Fast and simple

🕵️ Agent (Smart)

  • Uses multiple tools
  • Makes decisions
  • Multi-step reasoning
  • Web search, math, files
  • Complex problem solving
Agent TypeHow it Works
ReAct AgentReason + Act — thinks then acts step by step
OpenAI FunctionsUses OpenAI function calling feature
Plan & ExecuteMakes full plan first, then executes all steps
Self AskAsks itself follow-up questions to find answer
# ReAct Agent Example from langchain_core.tools import tool from langchain.agents import create_react_agent, AgentExecutor @tool def calculator(expression: str) -> str: """Calculate math expressions""" return str(eval(expression)) @tool def search_info(query: str) -> str: """Search for information""" return f"Info about: {query}" tools = [calculator, search_info] agent = create_react_agent(model, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True) result = executor.invoke({"input": "Calculate 25 * 4"})

2. What are Tools?

Definition

Tools are functions that Agents can use to perform actions — search web, calculate math, read files, get weather, and much more.

Tool TypeDescription
@tool decoratorSimple custom function as tool
StructuredToolTool with typed input schema
DuckDuckGoSearchBuilt-in web search tool
CalculatorBuilt-in math tool
PythonREPLRun Python code tool
FileSystemToolkitRead/write files toolkit
Agent Decision Flow: Question: "What is Python and calculate 10 * 5?" ↓ Agent THINKS: Need two tools! ↓ Action 1: search_info("Python") → "Python is programming language" Action 2: calculator("10 * 5") → "50" ↓ Final Answer: "Python is a programming language. 10 * 5 = 50" ✅

1. What are Embeddings?

Definition

Embeddings convert TEXT into NUMBERS (vectors) so computers can understand meaning. Similar words get similar numbers — like coordinates on a map.

How Embeddings Work: "King" → [0.2, 0.8, 0.5, ...] ← Similar vectors! "Queen" → [0.2, 0.7, 0.5, ...] ← Close to King! "Apple" → [0.9, 0.1, 0.3, ...] ← Far away! Steps: Text → Tokenize → Neural Network → Vector Numbers → Store in DB
# Embeddings with Sentence Transformers (Free!) from sentence_transformers import SentenceTransformer from sklearn.metrics.pairwise import cosine_similarity model = SentenceTransformer('all-MiniLM-L6-v2') sentences = ["I love AI", "AI is amazing", "I enjoy pizza"] embeddings = model.encode(sentences) similarity = cosine_similarity(embeddings) # Sentences 0 and 1 = HIGH similarity ✅ # Sentences 0 and 2 = LOW similarity ✅

2. What is RAG?

RAG = Retrieval Augmented Generation

R=Retrieval (Search & Find) + A=Augmented (Add to prompt) + G=Generation (AI generates answer)

FeatureNormal LLMRAG
KnowledgeTraining data onlyYour documents ✅
AccuracyMay hallucinate ❌Accurate ✅
UpdatesFixed knowledgeAlways current ✅
Private DataCannot access ❌Can access ✅
RAG Pipeline (2 Phases): PHASE 1 - INDEXING (once): Documents → Chunking → Embeddings → Vector Store PHASE 2 - RETRIEVAL + GENERATION (every question): Question → Embedding → Search Vector DB → Get Top K → Build Prompt (Context + Question) → LLM → Answer ✅

1. What is a Runnable?

Definition

A Runnable is a standard interface in LangChain that makes EVERYTHING work together using the same methods. Like a USB plug that fits any USB port — Runnable fits any LangChain component.

Runnable TypeWhat it DoesUse When
RunnableLambdaYour custom Python functionAdd custom logic
RunnablePassthroughPass input unchangedNeed original input later
RunnableParallelRun multiple chains at same timeSave time ⚡
RunnableSequenceRun one after anotherSteps depend on each other
RunnableBranchChoose which path to takeDifferent conditions
MethodWhat it DoesUse When
.invoke()Run once, get resultSingle input
.batch()Run many inputs at onceMultiple inputs ⚡
.stream()Get output word by wordLive typing effect
# RunnableParallel — Run 3 chains at same time! from langchain_core.runnables import RunnableParallel parallel_chain = RunnableParallel( explanation = explain_chain, examples = example_chain, quiz = quiz_chain ) result = parallel_chain.invoke({"topic": "Machine Learning"}) # Gets 3 results at the SAME time! ⚡

2. What is Structured Output?

Definition

Structured Output forces AI to return data in a specific organized format (JSON, Pydantic) instead of plain text paragraphs.

❌ Normal Output

  • Plain text paragraph
  • Random format
  • Hard to extract data
  • Cannot store in DB

✅ Structured Output

  • JSON / Pydantic format
  • Always same structure
  • Easy to extract data
  • Store in database easily
# Pydantic Structured Output (Best Way) from pydantic import BaseModel, Field class PersonInfo(BaseModel): name : str = Field(description="Person name") age : int = Field(description="Person age") city : str = Field(description="Person city") structured_model = model.with_structured_output(PersonInfo) result = structured_model.invoke("Ali is 30 from Karachi") print(result.name) # Ali print(result.age) # 30 print(result.city) # Karachi

Full LangChain Pipeline

DOCUMENTS (PDF, CSV, Web, Text) ↓ DOCUMENT LOADERS → load all docs ↓ TEXT SPLITTER → split into chunks ↓ EMBEDDING MODEL → convert to vectors ↓ VECTOR STORE → store vectors (FAISS/Chroma) ↓ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ USER ASKS QUESTION ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ↓ RETRIEVER → find relevant chunks ↓ PROMPT TEMPLATE → add context + question ↓ LLM MODEL → generate answer ↓ OUTPUT PARSER → format the answer ↓ FINAL OUTPUT ✅
